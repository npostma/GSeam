# Merge examples

Two single-tool Fusion 360 exports (`Simple1.ngc`, `Simple2.ngc`, both T3)
merged with `gseam.py`:

```bash
# default: one header/footer, renumbered, redundant same-tool toolchange
# skipped, nag comments removed
python3 ../../gseam.py Simple1.ngc Simple2.ngc output_no_options.ngc --no-tool-check

# with an explicit toolchange-subroutine call before every Tn M6
# (--keep-duplicate-toolchanges shown here so BOTH files get one)
python3 ../../gseam.py Simple1.ngc Simple2.ngc output_w_toolchange_routine.ngc \
    --no-tool-check --insert-toolchange-call --keep-duplicate-toolchanges

# SVG preview of the toolpaths/holes
python3 ../../gseam.py Simple1.ngc Simple2.ngc /tmp/out.ngc \
    --no-tool-check --preview-file preview_example.svg
```

`--no-tool-check` is used here because the example has no `tool.tbl`; on a
real machine leave it on so unknown tool numbers abort the merge.

Both output files and `preview_example.svg` in this folder are generated by
exactly these commands.
